# HuggingFace使用

## 一、`Hugging Face`生态概览
1. `Hugging Face`是一家提供开源`AI`工具和平台的公司，致力于简化预训练模型的使用，加速机器学习项目的开发与落地。最初以`Transformers`库闻名，该库极大地降低了使用`BERT`、`GPT`、`T5`等模型的门槛。如今，`Hugging Face`已发展成为一个完整的`AI`开发生态系统，支持自然语言处理、计算机视觉、语音处理、多模态任务等多个领域。
2. `Hugging Face`的生态系统主要由两个核心部分组成：
   - `Hugging Face Hub`：一个集中式的开源平台，用于托管和分享模型、数据集和应用
   - 工具链：提供了一套围绕预训练模型构建的工具库。这些组件彼此独立，又可以协同工作，覆盖了从数据处理到模型训练与推理的完整流程。
     - `Datasets`：用于加载和处理数据集的工作
     - `Tokenizers`：将文本转换为模型输入的工具
     - `Transformers`：用于加载、使用和微调各种预训练模型
3. 综上，`Hugging Face`是一套集成模型分享和模型使用各种工具的网站

## 二、预训练模型的加载与使用
1. 模型加载
   - `AutoModel`类
     - 作用：能够从本地或者远程直接加载模型，模型都是标准的预训练模型，不针对某一特定任务
     - 代码
       ```python
       from transformers import AutoModel
       # 加载模型，路径可以是远程也可以是本地
       model = AutoModel.from_pretrained("google-bert/bert-base-chinese")
       ```
     - 实际流程
       - 下载模型所需资源：远程下载模型配置文件和模型参数
       - 根据配置文件创建模型：根据下载的数据将模型读取到内存中
       - 加载模型权重：将训练好的模型权重参数加载到内存
   - `AutoModelForXXX`类
     - 作用：能够从本地或者远程直接加载模型，模型都是训练好的、针对某种特定任务的模型
     - 代码
       ```python
       # 加载模型
       from transformers import AutoModelForSequenceClassification
       model = AutoModelForSequenceClassification.from_pretrained("google-bert/bert-base-chinese")
       ```
     - 这类模型一般都有自己的特殊结构，例如上述模型中包含`BERT`的编码器主干和一个线性层用于输出每个类别的得分
2. 模型输入输出详解
   - 原生`Bert`模型
     - `forward()`方法输入
       ```text
       input_ids：词表中输入数据的索引
       attention_mask：掩码
       token_type_ids：生成第一个句子和第二个句子的索引，就是`Segment Embedding`这部分
       position_ids：位置编码
       head_mask：注意力模块的选定头部无效
       ```
     - `forward()`方法输出
       ```text
       last_hidden_state：最后一层输出的隐藏状态
       pooler_output：NSP任务的输出，在`CLS`token位置对应的输出，经过线性层(linear + tanh)进行处理得到的，相当于增强了提取能力的last_hidden_state[0]
       hidden_states：隐藏状态，中间每一层的隐藏状态，一般不用
       ```
   - 可以参考文档自己看参数的输入输出（参考[5]）

## 三、Tokenizer的加载与使用
1. 加载`Tokenizer`代码（加载流程与模型类似，从远程下载或者本地加载，并实例化对象）
   ```python
   from transformers import AutoTokenizer
   # 加载分词
   tokenizer = AutoTokenizer.from_pretrained("google-bert/bert-base-chinese")
   ```
2. 使用`Tokenizer`
   - `Tokenizer`都有哪些功能？
     - 子词切分
     - 编码映射
     - 添加特殊`token`
     - 截断与补齐
     - 生成辅助输入：主要是生成掩码`Mask`
   - `Tokenizer`常用的API有哪些？
     - 分词
       ```python
       tokens = tokenizer.tokenize("句子")
       ```
     - 转词表序号、序号转词表
       ```python
       ids = tokenizer.convert_tokens_to_ids(tokens)
       tokens = tokenizer.convert_ids_to_tokens(ids)
       ```
     - 编码：分词和转序号，并且添加特殊符号
       ```python
       tokenizer = AutoTokenizer.from_pretrained("./pretrained/bert-base-chinese")
       ids = tokenizer.encode("我爱自然语言处理")
       ```
     - 解码：解码会将一个`token ID`序列还原为对应的原始文本（或接近的文本）
       ```python
       string = tokenizer.decode(ids)
       ```
     - <font color'=yellow'>`tokenizer()`方法（即`__call__`）：直接构造模型的输入</font>
       ```python
       from transformers import AutoTokenizer
       tokenizer = AutoTokenizer.from_pretrained("./pretrained/bert-base-chinese")
       text = "我爱自然语言处理"
       # 编码文本为模型输入格式
       # 直接使用 tokenizer 作为方法调用
       inputs = tokenizer(text, return_tensors='pt')
       print(inputs)
       ```
3. 简单案例
   ```python
   texts = ["我爱自然语言处理", "我爱人工智能", "我们一起学习"]
   ## 创建模型和分词器
   model = AutoModel.from_pretrained("google-bert/bert-base-chinese")
   tokenizer = AutoTokenizer.from_pretrained("google-bert/bert-base-chinese")
   
   inputs = tokenizer(texts, return_tensors='pt', padding=True)
   with torch.no_grad():
      output = model(**inputs)
   
   print(output)
   ```

## 四、Datasets库的加载与使用
1. `datasets`是`Hugging Face`提供的一个轻量级数据处理库，专为自然语言处理任务设计，能够高效地支持模型训练流程中的数据加载与预处理操作。它的主要特点包括：
   - 加载方便：支持读取本地文件（如`CSV`、`JSON`），也支持加载在线公开数据集；
   - 结构清晰：数据集的内部结构类似表格，每条样本由若干字段组成；
   - 无缝协作：与`tokenizer`等`Hugging Face`模块高度集成，可直接构造模型输入；
   - 功能丰富：支持常见的数据处理操作，如批量映射`.map()`、字段筛选、训练/验证集划分`.train_test_split()`等。
2. 加载数据
   ```python
   dataset = load_dataset(format, data_files=路径或字典)
   # format: csv、json、parquet
   # data_files：文件路径
   ```
   ```python
   from datasets import load_dataset
   dataset_dict = load_dataset('csv', data_files={
       'train': './data/train.csv',
       'test': './data/test.csv'
   })
   # 访问的时候通过key + 索引进行访问
   ```
3. 预处理数据
   - 删除列
     ```python
     dataset = dataset.remove_columns(["cat"])
     ```
   - 过滤行
     ```python
     dataset = dataset.filter(lambda x: x["review"] is not None and x["review"].strip() != "" and x["label"] in [0, 1])
     ```
   - 划分数据集
     ```python
     dataset_dict = dataset.train_test_split(test_size=0.2)
     train_dataset = dataset_dict["train"]
     test_dataset = dataset_dict["test"]
     ```
   - 编码数据
     ```python
     # .map()是 datasets 中的核心方法之一，支持对整个数据集中的每一条样本或每一批样本进行统一处理，常用于文本编码（tokenizer）和数据字段换
     dataset = dataset.map(
             function,              # 要应用到每条样本上的函数（或每批样本上的函数）
             batched=False,         # 是否以“批”为单位处理样本；若为 True，则每次接收一个样本列表
             remove_columns=None    # 是否删除原始列，常用于清理不再需要的字段
     )
     ```
     ```python
     # 样例
     test_dataset = dataset_dict['test']
     tokenizer = AutoTokenizer.from_pretrained("google-bert/bert-base-chinese")
     
     def batch_encode(batch):
         # example:{label:[1,0,1,0],review:['******','******','******','******']}
         inputs = tokenizer(batch['review'],padding='max_length', max_length=128, truncation=True)
         inputs['labels'] = batch['label']
         return inputs
     
     test_dataset = test_dataset.map(batch_encode, batched=True, remove_columns=['label', 'review'])
     ```
4. 保存数据集
   - 提供了多种保存数据的格式方法
     - Arrow格式：高性能二进制类型`save_to_disk()`
     - CSV格式：`to_csv()`
     - JSON格式：`to_json()`
   - 加载数据
     - Arrow格式：高性能二进制类型`load_from_disk()`
     - CSV格式：`load_dataset()`
     - JSON格式：`load_dataset()`
5. 简单案例
   ```python
   train_dataset.set_format(
       type="torch",  # 指定输出为PyTorch张量
       columns=["input_ids", "attention_mask", "label"]  # 需要转换的字段
   )
   ```
