# 一些业务指标及汇报常用话术

## 一、业务指标
1. pv和uv
   1. uv：unique visitor，独立访客，有多少人来过，可以间接描述推广的效果
   2. pv：page view，页面浏览数量，可以间接描述内容的吸引力
2. 活跃数量
   1. DAU（Daily Active Users）：日活跃用户数量
   2. WAU（Weekly Active Users）：周活跃用户数量
   3. MAU（Monthly Active Users）：月活跃用户数量
3. 统计方向（选择合适的统计方向）
   1. 【加和】总量：功能上线后，使用总量达到xxx人次
   2. 【最大值】峰值：功能上线后，使用峰值达到xxx人次
   3. 【均值】平均：功能上线后，平均日活达到xxx人次
4. 其他指标
   1. 次留：次日留存率，即当天注册或活跃的用户在第二天再次活跃的比例，用于衡量用户粘性和产品效果
   2. 分享、点赞、转存、评论：常用指标
   3. 保存：保存xxx功能结果的数量
   4. 截图：截图记录xxx功能结果的
5. 使用方法：指标 + 统计方向 = 汇报话术
   

## 二、技术指标
1. SLA（Service-Level-Agreement）：服务稳定性
2. CT（Cost-Time）时间：接口调用时长
3. qps：每秒查询请求次数
4. tps：每秒事务请求次数
5. 成功率：xxx制作成功率

## 三、汇报常用话术
1. OKR话术：（稳定性、迭代效率、产品体验）
   1. 以技术赋能产品，提升服务稳定性，提升迭代效率，优化用户体验，助力业务快速发展
   2. 持续打磨智能化产品，为用户提供优质的产品体验，保证服务稳定性
   3. 聚焦智能化业务，高效支持业务迭代，保证服务稳定性

## 四、汇报模版和心得
1. 面向领导汇报注意事项
   1. Leader并不关注实现细节，关注的更多是带来了什么收益，带来了什么新的成绩，如何在未来长期保持
   2. 管理线Leader同时也关注如何评价工作指标，可量化且合适的指标确实是必要的
2. 总结模版基础版本
   ```txt
   一、年度OKR Review 
   # O1：XXX
   ## 1、完成情况自评：超出预期/符合预期/部分符合/完全不符合
   ## 2、核心量化指标进展
   ## 3、关键项目进展（项目数<=3,每个项目用200-300字描述）
   ### 项目一
   #### 背景和目标（最好附带交互图，一眼看出在xx地方实现了xx能力）
   #### 关键手段（如技术方案）
   #### 项目收益（核心指标和收益）和总结（个人思考）
   ### 项目二
   
   # O2：XXX
   ```
