8. Unsloth简介
   - Unsloth是一个能更高效地进行模型微调的框架，但是目前仅限于具有`CUDA`环境的设备，照原生的`transformers`训练模式能够节省很多空间
   - 样例case
     ```python
     from unsloth import FastLanguageModel
     import torch
     from trl import SFTTrainer
     from transformers import TrainingArguments
     from datasets import load_dataset
     max_seq_length = 2048 # Supports RoPE Scaling interally, so choose any!
     # Get LAION dataset
     url = "https://huggingface.co/datasets/laion/OIG/resolve/main/unified_chip2.jsonl"
     dataset = load_dataset("json", data_files = {"train" : url}, split = "train")
     
     # 4bit pre quantized models we support for 4x faster downloading + no OOMs.
     fourbit_models = [
         "unsloth/mistral-7b-bnb-4bit",
         "unsloth/mistral-7b-instruct-v0.2-bnb-4bit",
         "unsloth/llama-2-7b-bnb-4bit",
         "unsloth/gemma-7b-bnb-4bit",
         "unsloth/gemma-7b-it-bnb-4bit", # Instruct version of Gemma 7b
         "unsloth/gemma-2b-bnb-4bit",
         "unsloth/gemma-2b-it-bnb-4bit", # Instruct version of Gemma 2b
         "unsloth/llama-3-8b-bnb-4bit", # [NEW] 15 Trillion token Llama-3
         "unsloth/Phi-3-mini-4k-instruct-bnb-4bit",
     ] # More models at https://huggingface.co/unsloth
     
     # 加载模型
     model, tokenizer = FastLanguageModel.from_pretrained(
         model_name = "unsloth/llama-3-8b-bnb-4bit",
         max_seq_length = max_seq_length,
         dtype = None,
         load_in_4bit = True,
     )
     
     # Do model patching and add fast LoRA weights
     # 设置lora配置
     model = FastLanguageModel.get_peft_model(
         model,
         r = 16,
         target_modules = ["q_proj", "k_proj", "v_proj", "o_proj",
         "gate_proj", "up_proj", "down_proj",],
         lora_alpha = 16,
         lora_dropout = 0, # Supports any, but = 0 is optimized
         bias = "none",    # Supports any, but = "none" is optimized
         # [NEW] "unsloth" uses 30% less VRAM, fits 2x larger batch sizes!
         use_gradient_checkpointing = "unsloth", # True or "unsloth" for very long context
         random_state = 3407,
         max_seq_length = max_seq_length,
         use_rslora = False,  # We support rank stabilized LoRA
         loftq_config = None, # And LoftQ
     )
     
     # 训练器，处理数据
     trainer = SFTTrainer(
         model = model,
         train_dataset = dataset,
         dataset_text_field = "text",
         max_seq_length = max_seq_length,
         tokenizer = tokenizer,
         args = TrainingArguments(
             per_device_train_batch_size = 2,
             gradient_accumulation_steps = 4,
             warmup_steps = 10,
             max_steps = 60,
             fp16 = not torch.cuda.is_bf16_supported(),
             bf16 = torch.cuda.is_bf16_supported(),
             logging_steps = 1,
             output_dir = "outputs",
             optim = "adamw_8bit",
             seed = 3407,
         ),
     )
     # 开始训练
     trainer.train()
     
     # Go to https://github.com/unslothai/unsloth/wiki for advanced tips like
     # (1) Saving to GGUF / merging to 16bit for vLLM
     # (2) Continued training from a saved LoRA adapter
     # (3) Adding an evaluation loop / OOMs
     # (4) Cutomized chat templates
     ```

## 二、LoRA微调实战总结
1. 代码相关
   - `unsloth`微调代码：见`LLM/FineTuning/code&data/lora_unsloth`，使用`unsloth`进行模型微调，更高效节省空间
     - 加载模型：`FastLanguageModel.from_pretrained`
     - 加载LoRA配置：`FastLanguageModel.get_peft_model`
     - 执行训练：`SFTTrainer`
   - 原生`transformers`微调代码：见`LLM/FineTuning/code&data/lora`，使用`transformers`效率会更低
     - 加载模型：`AutoModelForCausalLM.from_pretrained`
     - 加载LoRA配置：`LoraConfig`
     - 执行训练：`SFTTrainer`
2. <font color='yellow'>微调经验相关，如何提高微调效果</font>
   - `epoch`一般是3轮，不用太多
   - `lora`的$r$、$\alpha$，适当可以调整，
   - `warmup_steps`：学习率预热，一开始的学习率很小，一般是总训练批次的10%左右
   - 追求精度的场景，慎重考虑量化
   - 一般微调能够有500个、1000多step就可以了，太多的话会降低模型本身的泛化能力
   - 学习率：大模型小学习率（14B、32B的-6或-5起步），小模型大学习率（8B的-4左右）
3. 评价指标
   - 论文中常用的`benchmark`
     - `MMCU`(`Massive Multitask Chinese Understanding`)，论文中用于对比不同大模型在中文知识、推理、学科能力上的优劣。
     - `BLEU`(`Bilingual Evaluation Understudy`)，机器翻译、文本摘要、对话生成、图像字幕等生成任务必用指标，快速对比模型效果。
     - `MMLU`(`Massive Multitask Language Understanding`)，即大规模多任务语言理解，是目前英文大模型领域最权威、最通用的综合能力基准测试，堪称`LLM`的 “通识高考”。
   - 企业中的评估往往是找人来评估真实答案和标准答案之间的差异，例如给一个表格，人工检查预测值和目标值之间的优劣来评价模型的好坏
4. <font color='yellow'>在真实企业中，微调并不经常使用</font>
   - 微调前期需要找训练数据集，这部分会消耗很多精力
   - 微调的效果不一定好，不一定比原来的预训练模型更好

## 四、使用`LLaMA-Factory`微调
1. `LLaMA-Factory`是什么？是一个简单易用且高效的大型语言模型（`Large Language Model`）训练与微调平台。通过`LLaMA Factory`，可以在无需编写任何代码的前提下，在本地完成上百种预训练模型的微调，框架支持：
   - 模型种类：`LLaMA`、`LLaVA`、`Mistral`、`Mixtral-MoE`、`Qwen`、`Yi`、`Gemma`、`Baichuan`、`ChatGLM`、`Phi`等等。
   - 训练算法：（增量）预训练、（多模态）指令监督微调、奖励模型训练、`PPO`训练、`DPO`训练、`KTO`训练、`ORPO`训练等等。
   - 运算精度：16比特全参数微调、冻结微调、`LoRA`微调和基于`AQLM/AWQ/GPTQ/LLM.int8/HQQ/EETQ`的`2/3/4/5/6/8`比特`QLoRA`微调。
   - 优化算法：`GaLore`、`BAdam`、`DoRA`、`LongLoRA`、`LLaMA Pro`、`Mixture-of-Depths`、`LoRA+`、`LoftQ`和`PiSSA`。
   - 加速算子：`FlashAttention-2`和`Unsloth`
   - 推理引擎：`Transformers`和`vLLM`
   - 实验监控：`LlamaBoard`、`TensorBoard`、`Wandb`、`MLflow`、`SwanLab`等等。
2. `LLaMA-Factory`下载与环境配置
   ```python
   # 下载项目LLaMA-Factory
   git clone https://github.com/hiyouga/LLaMA-Factory.git
   # 创建环境
   conda create -n llama_factory python=3.10
   conda activate llama_factory
   # 安装各种依赖
   cd /xxx/LLaMA-Factory
   # 安装 Mac 专用 PyTorch（不带 CUDA！）

   llamafactory-cli version
   ```
3. `LLaMA-Factory`使用
   - 启动服务
     ```python
     # 启动服务
     llamafactory-cli webui
     ```
   - 在界面上选择自己需要的微调方式
     - 模型：模型名称、加载路径、加载平台
     - 微调方法：`LoRA`、`full`等
     - 量化等级：选择对应的量化参数
     - 训练数据选择：`LLaMA-Factory`目前支持`Alpaca`格式和`ShareGPT`格式的数据集。我们自己整理好格式的数据需要添加到数据集信息中
     - 训练学习率
     - `LoRA`微调设置，比如秩的大小和缩放程度
