# 《从零构建大模型》学习笔记

1. 较小的批次大小会减少训练过程中的内存占用，但同时会导致模型更新时产生更多的噪声。批次大小同样是训练大语言模型时需要仔细权衡并尝试调整的超参数
2. 嵌入层与线性层
   - 嵌入层更多的是从嵌入层矩阵中根据索引找到对应的向量
   - 线性层是标准的矩阵乘法和矩阵加法，是完全不一样的
3. 位置编码策略
   - 分类：主要包括相对位置编码和绝对位置编码
     - 绝对位置编码：`Transformer`采用正余弦编码方式就是绝对位置编码，`GPT`采用的是可学习的位置编码
     - 相对位置编码：
   - 位置编码常用算法一览
   
   |    类型    |        代表方法        |     核心思想      |      实现方式       |      典型模型       |
   |:--------:|:------------------:|:-------------:|:---------------:|:---------------:|
   | 绝对位置编码	  |       正弦余弦编码       | 	编码 “第几个位置” 	 |   	位置向量 + 词嵌入   | 原始 Transformer  |
   | 绝对位置编码 	 |      可学习位置嵌入	      |  编码 “第几个位置”	  |  可学习向量 + 词嵌入	   |  BERT、RoBERTa   |
   | 相对位置编码	  | Transformer-XL RPE |  	编码 “相距多远”	  |  注意力中加入相对位置向量   | 	Transformer-XL |
   | 相对位置编码	  |       T5 RPE       |  	编码 “相距多远”	  |  注意力分数加相对位置偏置   |       	T5       |
   | 相对位置编码	  |       RoPE	        |  编码 “相对角度”	   | Query/Key 向量旋转	 |   GPT-3、LLaMA   |
   | 相对位置编码	  |       ALiBi        |  	编码 “距离衰减”	  |   注意力分数加线性偏置	   |    MPT、BLOOM    |
4. `pytorch`实现的`softmax()`与基本的`softmax()`存在部分不同
   - `pytorch`实现的`softmax()`进行了相关优化，保证在处理大输入值和小输入值时不会趋向于无穷或者零
     $$\text{Softmax}(x_i) = \dfrac{e^{x_i - \max(x)}}{\sum_{j} e^{x_j - \max(x)}}$$
   - 基本的`softmax()`没有进行相关优化
5. 掩码机制的使用：理论上来说，掩码计算的注意力机制，面对未来的、需要掩码的`token`，我们应该使用负无穷来代替，这样经过`softmax`后获得的权重系数就是零了
6. 模型困惑度（PPL，`perplexity`）
   - 定义：模型负对数似然损失的指数形式
     $$
     PPL = \exp\left( -\frac{1}{N}\sum_{i=1}^N \log P(w_i \mid w_{<i}) \right)
     $$
   - 用途：用于提供一种更容易解释的方式来理解模型在预测序列中的下一个词元时的不确定性
   - 交叉熵、KL散度、模型困惑度
     - 基本定义
       - 交叉熵
         $$H(p,q) = -\sum p\log q$$
       - KL散度
         $$D_{\text{KL}}(p\parallel q) = \sum p\log\frac{p}{q}$$
       - 模型困惑度
         $$\text{PPL} = \exp\left( -\frac{1}{N}\sum_{i=1}^N \log P(w_i \mid w_{<i}) \right)$$
     - 区别对比
   
       |   指标    |     核心含义      |      主要用途      |       特点       |
       |:-------:|:-------------:|:--------------:|:--------------:|
       |   交叉熵   | 真实与模型分布的编码代价  |    模型训练损失函数    |  可直接求导、适合反向传播  |
       |  KL散度   |  两个概率分布的差异距离  | 理论分析、VAE、分布对齐  | 非负、不对称，表征分布差距  |
       | PPL困惑度  |  语言模型预测不确定度   |     模型评测指标     | 越小越好，直观表征预测难易  |
   - 总结
     - 监督学习标签固定，**KL与交叉熵优化等价**
     - **PPL 是交叉熵损失的指数形式**，仅用于语言模型评测
     - 训练用交叉熵、理论分析用KL、模型打分用PPL
7. 温度缩放 & `Top-K`采样
   - 温度缩放
     - 定义：将模型最后输出给线性层的结果统一除以一个大于零的数（这个数本身就是温度）
       - 除以的数大于1，会导致词元分布的概率更均匀
       - 除以的数小于1，会导致词元概率分布更加自信（更尖锐或陡峭）
     - 优点：通过温度缩放，可以控制模型最终输出的`token`的多样性，允许模型探索概率较低但是可能更具创造性和趣味性的生成
     - 缺点：有时候会导致完全不正确或者无意义的输出
   - Top-K采样
     - 定义：获取下一个词元的时候，将输出限制在可能性最高的K个选择中
   - 场景推演
     - 正式场合：使用较低的温度缩放 + 较低的K值
     - 非正式场合：使用较高的温度缩放 + 较大的K值
8. 分类微调和指令微调
   - 分类微调：模型呗训练来识别一组特定的类别标签
   - 指令微调：指令微调是指使用特定的指令数据对一组任务进行训练，以提高语言模型理解和执行自然语言提示此中描述任务的能力
     ```txt
     - 原始数据集
     {
        "instruction": "Indentify the correct spelling of the following word.",
        "input": "Ocassion",
        "output": "The correct spelling is 'Occasion'."
     }
     
     - 模版
     Below is an instruction that describes a task. Write a response that...
     ### Instruction: ....
     ### Response: ....
     ```
